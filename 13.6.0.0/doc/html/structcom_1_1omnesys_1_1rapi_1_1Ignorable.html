Detailed Description

Represents an object whose underlying type is a value type that has an associated state indicating whether the value should be ignored, cleared, or used.

Ignorable<> is designed to convey three states that the template type cannot convey on their own. The three states are Ignore, Clear and Use.

Ignorable<> is used in PnlInfo, which contains a number of properties. However, these properties are not always updated at the same time.
Ignorable<> allows you to test whether or not you should ignore the property, and then if you shouldn't ignore it, what to do with the value (clear it or use it).

For example, when you have a position, your open pnl changes as the market moves, but the closed pnl does not. During this time, you may receive PnlInfo updates indicating that you should 'Ignore' PnlInfo.ClosedPnl (because the value you already have remains unchanged) while 'Use'-ing PnlInfo.OpenPnl (because the market moved, and there is now a new open pnl).

Continuing this example, while you have the open position, you will have a well defined avg open fill price. As with the closed pnl value, the avg open fill price does not change with the market, so you may receive PnlInfo updates with new OpenPnl values, but 'Ignore'-able AvgOpenFillPrice values (because the value you currently have remains unchanged).

However, when the position is closed, you will get an update where PnlInfo.AvgOpenFillPrice.Clear is true. This is because the value is no longer well defined, so you should clear out whatever value you might have.
