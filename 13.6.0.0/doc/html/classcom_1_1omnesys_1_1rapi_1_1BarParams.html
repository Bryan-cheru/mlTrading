Detailed Description

This class contains the parameters for defining a data bar.

The BarParams class is used to define bars when subscribing to the real time data stream or when requesting historical data.

◆ CustomSession

bool CustomSession
getset

Data bars typically use market mode transitions to determine when a trading session begins and ends.

These session events can cause bars to be opened or closed. To configure a custom session time, this flag must be set to true. Custom sessions can only be applied to tick, volume and range bars.

Custom sessions may be ignored for tick bars where the tick size is one.

◆ Profile

bool Profile
getset

This flag indicates whether or not to include volume/data profile information.

Currently, only replayed minute bars and multi-tick tick bars support volume/data profile information. This flag is ignored where there is no support. Retrieving volume profile is generally slower than not retrieving the data.

Volume profile information is the volume traded at each price level within the bar's price range. The volumes are separated into three categories - volumes where there was no aggressor, volumes where the buyer was the aggressor, and finally volumes where the seller was the aggressor.

The number of trades is also included for each of these categories.
