Detailed Description

This class contains the parameters for an order.

This class must be used with REngine::sendOrderList() to utilize all available order parameters.

A value of "yes" enables this feature; otherwise leave empty.

◆ UserTag

string UserTag
getset

This optional value specifies a user tag that is persisted with the resultant order by the infrastructure.

If specified, this value will be present on subsequent order updates published by the infrastructure.

If this value is empty or null, no user tag will be sent.

This value differs from OrderParams.UserMsg in that it cannot be updated with subsequent order operations.

This value differs from OrderParams.Tag in that it is not used by R | Trader. The intent of UserTag is to allow an end user of R | Trader (or perhaps any application written on top of R | API) to specify a tag that is visible to other clients of the trading platform.

Optional.

This optional value specifies the window or widget name from which this new order request originated.

If specified, this value can appear on subsequent order updates in either the WindowName or OriginatorWindowName field. When placing a list of orders, only the first value of WindowName will be used for all orders in the list.
